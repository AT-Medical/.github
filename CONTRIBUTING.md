# Contributing Guidelines

## 🇩🇪 Beitragsrichtlinien

Alle Änderungen an AT Medical Repositories folgen diesem Workflow.

### Workflow

1. Feature-Branch erstellen
2. Pull Request öffnen
3. Review anfordern
4. Nach Genehmigung mergen

Direkte Commits in den Main-Branch sind nicht erlaubt.

### Sicherheit

Folgendes darf niemals eingecheckt werden:

- Passwörter
- API-Schlüssel
- Tokens
- Zertifikate
- Interne URLs oder Zugangsdaten

### Dokumentation

- Strukturelle Änderungen müssen dokumentiert werden
- README-Dateien müssen zweisprachig sein (Deutsch zuerst, dann Englisch)
- Neue Repositories müssen die Governance-Standards einhalten

### Code-Qualität

- Explizite und lesbare Implementierungen bevorzugen
- Nur angeforderte Änderungen vornehmen
- Unnötiges Refactoring vermeiden

---

## 🇬🇧 Contributing Guidelines

All changes to AT Medical repositories follow this workflow.

### Workflow

1. Create a new branch
2. Open a Pull Request
3. Request review
4. Merge after approval

Direct commits to the main branch are not allowed.

### Security

Never commit:

- passwords
- API keys
- tokens
- certificates
- internal URLs or credentials

### Documentation

- Structural changes must be documented
- README files must be bilingual (German first, then English)
- New repositories must follow governance standards

### Code Quality

- Prefer explicit and readable implementations
- Only make requested changes
- Avoid unnecessary refactoring
