# Support

## 🇩🇪 Support & Kontakt

### Wie erhalte ich Unterstützung?

Bei Fragen oder Problemen mit Repositories und Diensten der AT Medical GmbH® stehen folgende Kontaktmöglichkeiten zur Verfügung:

**Technische Anfragen**

Bitte nutzen Sie unser Kontaktformular:  
[www.at-medical.de/kontakt](https://www.at-medical.de/kontakt)

**E-Mail**  
[Info@at-medical.de](mailto:Info@at-medical.de)

**Sicherheitsrelevante Meldungen**

Für die Meldung von Sicherheitslücken oder Schwachstellen lesen Sie bitte die [SECURITY.md](SECURITY.md).  
Bitte öffnen Sie **keine** öffentlichen GitHub-Issues für sicherheitsrelevante Themen.

### GitHub Issues

Öffentliche GitHub-Issues sind für technische Fehlerberichte und Feature-Anfragen vorgesehen, die für das jeweilige Repository relevant sind.

Issues, die vertrauliche, personenbezogene oder sicherheitsrelevante Informationen enthalten, werden geschlossen und direkt bearbeitet.

### Reaktionszeiten

Wir bemühen uns, Anfragen zeitnah zu bearbeiten. Eine Reaktionszeit kann nicht garantiert werden, da diese Repositories im Rahmen einer strukturierten Entwicklungsumgebung betrieben werden.

## 🇬🇧 Support & Contact

### How do I get support?

For questions or issues related to AT Medical GmbH® repositories and services, the following contact options are available:

**Technical Requests**

Please use our contact form:  
[www.at-medical.de/kontakt](https://www.at-medical.de/kontakt)

**Email**  
[Info@at-medical.de](mailto:Info@at-medical.de)

**Security-Related Reports**

To report security vulnerabilities or weaknesses, please read [SECURITY.md](SECURITY.md).  
Please do **not** open public GitHub issues for security-related matters.

### GitHub Issues

Public GitHub issues are intended for technical bug reports and feature requests relevant to the respective repository.

Issues containing confidential, personal or security-relevant information will be closed and handled directly.

### Response Times

We aim to address requests in a timely manner. A guaranteed response time cannot be committed to, as these repositories are operated as part of a structured development environment.
