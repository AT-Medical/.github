# Security Policy

## Reporting a Vulnerability

The security of AT-Medical software and infrastructure is important to us.

If you discover a security vulnerability in one of our repositories, please report it responsibly.

Do not open a public GitHub issue for security vulnerabilities.

Instead, please contact us directly:

Email: security@at-medical.example

Include the following information if possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested mitigation or fix (if available)

We will acknowledge your report as soon as possible and work to resolve the issue.

## Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate and fix the vulnerability
- Do not publicly disclose the issue until it has been addressed
- Avoid accessing or modifying user data

## Supported Versions

Security updates are applied to the most recent versions of our software and actively maintained repositories.

Older versions or archived repositories may not receive security updates.

## Security Updates

When a vulnerability is confirmed:

- A fix will be developed
- A security update will be released
- Relevant repositories will be updated accordingly

## Scope

This policy applies to all repositories under the AT-Medical GitHub organization unless a repository provides its own security policy.

## Thank You

We appreciate responsible disclosure and thank you for helping improve the security of our systems.
