# Security Policy

## 🇩🇪 Sicherheitsrichtlinie

### Sicherheitslücken melden

Die Sicherheit der Software und Infrastruktur von AT Medical ist uns wichtig.

Wenn Sie eine Sicherheitslücke in einem unserer Repositories entdecken, melden Sie diese bitte verantwortungsvoll.

Bitte öffnen Sie **kein** öffentliches GitHub-Issue für Sicherheitslücken.

Kontaktieren Sie uns stattdessen direkt:

E-Mail: [Info@at-medical.de](mailto:Info@at-medical.de)

Geben Sie wenn möglich folgende Informationen an:

- Beschreibung der Sicherheitslücke
- Schritte zur Reproduktion des Problems
- Mögliche Auswirkungen
- Vorgeschlagene Maßnahmen oder Lösungen (falls vorhanden)

Wir bestätigen Ihren Bericht so bald wie möglich und arbeiten an einer Behebung.

### Verantwortungsvolle Offenlegung

Wir bitten Sie:

- uns ausreichend Zeit zu geben, die Sicherheitslücke zu untersuchen und zu beheben
- das Problem nicht öffentlich bekannt zu machen, bevor es behoben wurde
- keinen Zugriff auf Benutzerdaten zu nehmen oder diese zu verändern

### Unterstützte Versionen

Sicherheitsupdates werden für die aktuellen Versionen unserer Software und aktiv gepflegten Repositories bereitgestellt.

Ältere Versionen oder archivierte Repositories erhalten möglicherweise keine Sicherheitsupdates.

### Sicherheitsupdates

Wenn eine Sicherheitslücke bestätigt wird:

- wird ein Fix entwickelt
- wird ein Sicherheitsupdate veröffentlicht
- werden betroffene Repositories entsprechend aktualisiert

### Geltungsbereich

Diese Richtlinie gilt für alle Repositories unter der AT Medical GitHub-Organisation, sofern ein Repository keine eigene Sicherheitsrichtlinie bereitstellt.

### Danke

Wir schätzen verantwortungsvolle Meldungen und danken Ihnen für Ihren Beitrag zur Sicherheit unserer Systeme.

## 🇬🇧 Security Policy

### Reporting a Vulnerability

The security of AT Medical software and infrastructure is important to us.

If you discover a security vulnerability in one of our repositories, please report it responsibly.

Do not open a public GitHub issue for security vulnerabilities.

Instead, please contact us directly:

Email: [Info@at-medical.de](mailto:Info@at-medical.de)

Include the following information if possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested mitigation or fix (if available)

We will acknowledge your report as soon as possible and work to resolve the issue.

### Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate and fix the vulnerability
- Do not publicly disclose the issue until it has been addressed
- Avoid accessing or modifying user data

### Supported Versions

Security updates are applied to the most recent versions of our software and actively maintained repositories.

Older versions or archived repositories may not receive security updates.

### Security Updates

When a vulnerability is confirmed:

- A fix will be developed
- A security update will be released
- Relevant repositories will be updated accordingly

### Scope

This policy applies to all repositories under the AT Medical GitHub organization unless a repository provides its own security policy.

### Thank You

We appreciate responsible disclosure and thank you for helping improve the security of our systems.
